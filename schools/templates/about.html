About sgschools.today

I started sgschools.today because I felt that students of computer science (and computer engineering) should be exposed to geographic information and geographic information systems (GIS), and were not. I drew up the following list of concepts, software, frameworks and websites, which I felt were important to anyone interested in creating an app or website that has a map:

  • longitude and latitude
  • geographic information systems
  • geolocation and reverse geolocation
  • OpenStreetMap and other projects involving volunteered geographic information (VGI)
  • coordinate systems like WGS84 and OSGB36
  • Django and Geodjango
  • PostgreSQL and PostGIS
  • Leaflet.js

The best way to get students to tinker or experiment with geographic information is to get them working on a Critical Inquiry project, and so, I proposed a CI project with this topic for Semester 2 (2018/19 Academic Year). Three students from the MSc (Information Systems) programme took up the challenge, and this site is the product of their 13-week effort.

Phase 1: Laying the Foundations

In this phase, the data model was developed, a basic map interface developed to show the location of kindergartens and primary schools in Singapore. The focus was not on a rich set of functionality, but rather, to get a foundational set of code which could be built upon by future groups of students taking the Critical Inquiry module. Phase 2: Adding More Content

OneMap API, OpenCage API and Query UI were used to implement search by address / postal code. Implemented caching, so that all the schools are stored in memory when the Django application is started.
